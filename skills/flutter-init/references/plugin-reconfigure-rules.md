# Plugin Reconfigure Rules

## 用途

- 规定 `flutter-init` 遇到 `--force` 时如何处理插件重配
- 保证插件重配只是一个显式动作，不会默认覆盖已有配置
- 保证插件处理完成后继续后续初始化任务，而不是停在插件步骤

## `--force` 语义

- `--force` 表示重新配置当前 RD 范围内的插件
- 没有 `--force` 时，如果插件尚未配置，则默认执行首次插件配置
- 没有 `--force` 时，如果插件已经配置，则默认保留当前已有插件配置
- `--force` 只影响插件重配，不改变 DDD 结构、模块规划、依赖治理这些常规流程

## 处理规则

### 传入 `--force`

- 重新检查当前项目已有插件接入状态
- 对当前 RD 范围内需要的插件重新执行配置
- 刷新相关平台配置、项目引用、初始化入口或环境变量占位
- 插件重配完成后，继续执行后续初始化或补全任务

### 未传 `--force`

- 先检查当前 RD 范围内所需插件是否已经存在
- 如果插件尚未配置，执行首次插件配置
- 如果插件已经配置，跳过插件重配并保持已有配置不变
- 直接继续后续初始化或补全任务

## 适用对象

这里的插件包括但不限于：

- Firebase 相关插件
- 崩溃监控与埋点插件
- 图片、媒体、文件选择插件
- 推送、深链、实时通信插件
- 任何需要额外平台配置或初始化接线的第三方 SDK

## 输出要求

执行后必须说明：

- 是否检测到 `--force`
- 哪些插件被重配
- 哪些插件是首次配置
- 哪些插件保持不变
- 插件步骤结束后继续了哪些后续任务
