# Draft Completeness Gate

Treat a design as complete enough for review only when the reviewer can judge it as a real product surface instead of a mood fragment.

## Minimum Evidence

Review the draft normally when all of these are visible or explicitly specified:

- the main screen, frame, or page family is visible
- the primary CTA is visible
- the typography ladder can be read
- text-to-surface contrast can be judged
- the main information hierarchy is visible
- at least one important non-happy-path clue exists, or the state plan is attached
- the business moment or trust moment is visible enough to critique

## Not Complete Yet

Return `blocked` when the input is mostly one of these:

- a text-only brief
- a decorative hero with no usable UI structure
- a cropped fragment with no CTA or hierarchy context
- a single perfect state with no state plan and no clear production scope
- an unfinished Pencil structure with no readable screen output

## Review Trigger

In the workflow, “design draft complete” usually means one of these:

- a shared visual draft or preview pack is ready for freeze discussion
- a module page draft is ready for page-level freeze discussion
- a Pencil rebuild is visually complete enough for parity or handoff review

Do not wait for code. This skill reviews design drafts, not implemented UI.
