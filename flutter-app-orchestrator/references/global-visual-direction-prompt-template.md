# 全局视觉方向选择提示词模板

在生成前，先将本模板填充并保存为 `docs/design/prompts/global-visual-direction-prompt.md`。该正式提示词文件必须包含：模板来源、PRD 与市场分析来源文件、逐项“PRD 要求或市场发现—提示词表达—来源位置”映射、产品设计原则检查、当前视觉表达预算预设（含三方向分布与评分加权）、以及自行选择的代表页面和 PRD 依据。它用于定义三套方向的比较约束，不得直接作为代表页面效果图的生成提示词。选择代表页面时，应优先覆盖最高优先级用户任务、高价值时刻和最能区分视觉方向的信息；使用 `page-hifi-mockup-prompt-template.md` 为每套候选方向填充生成内容，但无需保存独立页面提示词文件。全局探索产生的代表页面效果图只用于比较和选择视觉方向，不得作为该页面或任何后续页面的效果图确认、页面设计冻结或实现批准依据；进入模块交付后，当前页面仍须按页面级流程生成、评审并确认其效果图。只填写能够在 `product-brief.md`、`mvp-scope.md`、`user-stories.md`、`user-flows.md`、`screen-spec.md`、`market-analysis.md` 或已批准决策中追溯的信息；不得补造功能、用户目标、业务承诺或页面内容。三套方向必须使用同一代表页面、设备尺寸、必需内容、文案、数据、用户任务和页面状态；允许布局构图、内容分组、视觉节奏、组件轮廓、图像、材质和动效意图随方向变化，但不得改变产品含义。必须遵守产品 brief 中的视觉表达预算预设：任务清晰为硬门槛，表达上限与探索偏置随预设调整，禁止三套方向都收敛为中性克制的专业风。

## 二、探索目标

请基于 PRD 和 `docs/product/market-analysis.md`，推荐 3 套差异明确、适合生成效果图比较的视觉方向。每套方向都必须说明它如何服务产品核心任务、符合或有意识地区别于品类惯例，并回应市场分析中的机会或风险。

三套方案必须保持以下内容一致：

* 产品定位一致
* 页面功能和信息结构一致
* 核心用户任务一致
* 示例文案和数据一致
* 设备尺寸和页面数量一致

只改变以下设计变量：

* 色彩体系
* 字体气质
* 圆角与形状语言
* 卡片和容器风格
* 图像或插画风格
* 图标风格
* 留白与信息密度
* 阴影、边框和材质
* 品牌情绪
* 动效想象

三套方向不能只是更换颜色，必须在视觉语言和品牌感知上形成明显区别。

## 三、视觉方向建议

请基于产品特点、PRD、市场分析和视觉表达预算自行定义三套方向，方向类别不受以下示例限制，但必须满足上方“本轮三方向强制分布”。可以参考但不要机械照搬以下类型：

方向 A：
可信、理性、专业、高信任感。

方向 B：
温暖、亲和、生活方式化、有情绪价值。

方向 C：
前沿、智能、具有鲜明数字产品感与可复述签名。

如上述方向不适合产品，必须重新定义三套更合理的方向；不得为了凑数沿用示例类别；不得让三套都落在中性克制的专业风。

## 四、每套方向必须说明

为每个视觉方向输出以下内容：

### 1. 方向名称

使用一个容易记忆的中文名称，并附一条简短英文名称。

### 2. 核心概念

用 2—3 句话说明该方向表达的品牌感受，以及为什么适合目标用户。

### 3. 视觉关键词

提供 5—8 个具体关键词。

不要只使用“高级、简约、现代”等宽泛词汇，应使用更具画面感的描述，例如：

* 编辑式排版
* 柔和自然光
* 精密数据仪表
* 低饱和矿物色
* 高对比黑白界面
* 纸张质感容器
* 紧凑型信息面板

### 4. 色彩系统

说明：

* 主背景色
* 容器色
* 主文字色
* 辅助文字色
* 品牌主色
* 功能强调色
* 成功、警告和错误状态色
* 是否使用渐变

给出建议的 HEX 色值。

### 5. 字体与排版

说明：

* 标题字体气质
* 正文字体气质
* 数字字体表现
* 字号层级
* 字重使用
* 字间距
* 对齐方式
* 页面整体信息密度

### 6. 形状语言

说明：

* 卡片圆角
* 按钮圆角
* 输入框样式
* 图标容器
* 边框粗细
* 分割线风格
* 阴影方式
* 是否使用玻璃拟态或半透明材质

### 7. 图像与图标

说明：

* 使用摄影、插画、3D 图像还是纯图形
* 人物或产品图像的拍摄风格
* 图标是线性、面性还是混合型
* 图片裁切方式
* 图片与界面的融合方式

### 8. 核心组件表现

描述以下组件在该方向中的设计方式：

* 主要按钮
* 次要按钮
* 信息卡片
* 数据模块
* 输入框
* 标签
* 顶部导航
* 底部导航
* 弹窗
* 空状态

### 9. 适用性分析

说明：

* 最适合哪类用户
* 最适合什么商业定位
* 该方向的主要优势
* 可能存在的风险
* 扩展到完整 App 后是否容易保持一致

## 五、视觉效果图要求

请为每个方向生成同一个代表性核心页面，以便公平比较。

代表页面：
[例如：首页 / 数据总览页 / AI 对话页 / 课程详情页]

页面必须包含：
[列出该页面固定出现的模块]

每个方向只生成一张完整页面。

输出要求：

* 三套方案使用完全相同的信息结构和文案
* 同一设备尺寸
* 同一页面状态
* 正视角
* 高保真移动端 UI
* 完整显示状态栏和底部区域
* 不使用手机模型
* 不使用手持设备
* 不使用透视
* 不放置在桌面或现实场景中
* 不制作 Behance 展示板
* 不在一个方向中混合多种视觉风格

## 六、评选建议

生成三套方向后，请从以下维度进行对比评分，每项满分 10 分；按上方“评分加权强调”提高对应维度的比较权重，但不得用加权牺牲任务清晰与信息可读性：

* 品牌辨识度
* 目标用户匹配度
* 信息可读性
* 产品可信度
* 情绪吸引力
* 功能扩展能力
* 设计系统可维护性
* 商业化适配能力
* 市场差异化机会
* 品类用户预期匹配度
* 首次尝试与信任支持度
* 视觉签名可复述性

最后列出三套候选方向、评分和各自的主要优势与风险。

不要推荐、选择或确认任何方向。请明确等待用户选择并确认一套方向，或要求继续迭代。选定后，若 Global Freeze Signature Rule 适用，必须确认该方向是否有可复述的视觉签名；答案为否时不得冻结，应继续迭代。若规则不适用，记录 N/A 原因。用户可声明 pin / raise / loosen 覆盖表达预算。

优先考虑真实产品的长期一致性和可用性。表达强度跟随预设，不要默认追求概念图冲击力，也不要默认把“克制”当成高级。
